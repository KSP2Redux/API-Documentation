    Class Universe

    The central orchestrator for Lua-based patching: loads patch scripts, registers patches and new assets, sorts stages topologically, and runs the resulting pipeline against each patched asset.

    Inheritance
    object
    Universe
    Namespace: PatchManager.LuaPatching
    Assembly: Assembly-CSharp.dll
    Syntax
    public class Universe
    Remarks

    Converter and submodule discovery is performed by LuaPatchingModule at module-init time, which populates Converters and SubmoduleTypes by scanning every loaded assembly. Each universe instance then constructs its own PatchManagerCore and live submodule instances, exposes them as the global PM table, and tracks per-mod patch state. Mods load Lua patches via the LoadPatch* methods, which are expected to register patches and stages through PM; once loading is done, SetupPatchesForRun() finalizes ordering and the RunAllPatchesFor overloads execute the chain against each asset.

    string name

    The asset's addressables address.
    JToken data

    int patchCount

    int errorCount

    Set to the number of patches that threw.
    Returns
    Type Description
    JToken

    The patched JSON, or null when the asset was removed.
    Remarks

    Patches are chained: each one operates on the previous patch's MoonSharp.Interpreter.DynValue when the converters match, otherwise the chain is flushed back to JSON, lifted by the new converter, and chaining resumes. Returns data unchanged when no patch applied; returns null when a patch removed the asset.
