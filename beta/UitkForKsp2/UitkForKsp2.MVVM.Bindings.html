    Namespace UitkForKsp2.MVVM.Bindings

    Classes

    CommandBinding

    Convenience binding for Button.clicked -> ICommand. Wires the button's clicked event and enable state to a parameterless command on the data source.

    UXML usage:

    <ui:Button name="buy-btn" text="Buy">
    <Bindings>
        <CommandBinding property="text" data-source-path="BuyCommand" />
    </Bindings>
</ui:Button>

    The "property" attribute is required by the infrastructure but unused - set it to any valid property (by convention "name"). For non-Button elements, use EventBinding.

    CommandBinding.UxmlSerializedData

    EventBinding

    Binds any UI Toolkit event on any VisualElement to an ICommand or ICommand<TEvent> on the data source. Works with every EventBase subclass - clicks, pointer, keyboard, focus, change, drag, navigation, and more.

    UXML usage:

    <!-- Parameterless: event fires command, data discarded -->
<EventBinding property="name" data-source-path="HoverCommand" event-type="PointerEnterEvent" />

<!-- Typed: ViewModel receives full event object -->
<EventBinding property="value" data-source-path="KeyDownCommand" event-type="KeyDownEvent" />

<!-- ChangeEvent special case: extracts newValue automatically -->
<EventBinding property="value" data-source-path="VolumeCommand" event-type="ChangeEvent&lt;System.Single&gt;" />

    The "property" attribute is required by the infrastructure but unused - set it to any valid property (e.g. "name"). Reflection runs only once per command instance - subsequent updates only sync enabled state.

    EventBinding.UxmlSerializedData

    EventTypeResolver

    Resolves event type name strings from UXML attributes to System.Type objects. Supports simple names ("KeyDownEvent") and generic names ("ChangeEvent<System.Single>"). Results are cached so reflection runs only once per unique type string.
